# Video Lens V1 Spec

## 目标与运行形态

Video Lens（中文名“片语”）是运行在 Tampermonkey 中的 Bilibili / YouTube 视频总结 userscript。项目使用 pnpm、Vite、TypeScript 和 `vite-plugin-monkey`，UI 由原生 DOM 与 Shadow DOM 构成。

支持页面：

- Bilibili `/video/`、`/list/`
- YouTube `/watch?v=...`、`/shorts/...`

## 启动与总结流程

页面加载后先显示可拖动悬浮按钮。用户点击按钮时：

1. 展开摘要和对话面板。
2. 尝试恢复当前视频、模型、Prompt、语言和字幕源匹配的摘要缓存。
3. 命中缓存时直接展示，不发送模型请求。
4. 未命中时读取字幕并立即调用文本模型总结。

悬浮吉祥物与 Panel 之间使用约 200ms 的双向缩放/淡入淡出过渡；关闭时 Panel 先缩回，再显示吉祥物。左右位置使用各自靠外侧的变换原点，减少动态效果偏好开启时取消动画和等待。

摘要支持流式输出、长字幕分块/合并、Markdown 安全子集渲染、复制和 Markdown 导出。模型 reasoning 在初始摘要和后续问答中进入同一折叠区：流式时展开显示 `Thinking`，正文开始或请求完成后折叠为 `Thought for <duration>`。手动“重新生成”仍会重新执行总结流程。摘要完成后，可在下方对话框进行基于当前视频内容的连续问答。

长字幕分段使用并发上限 2，并按原字幕索引保存结果。分段阶段只更新完成进度，不通过 `onDelta` 渲染分段正文。空分段自动重试一次，第二次仍为空则明确失败；整体合并结果为空、等同于单个分段或仍带明显的 `第 1/3 段` 标记时自动重试一次，第二次仍不完整则报错，不用分段内容冒充最终摘要。视频卡片通过统一 `VideoInfo.stats` 显示 provider 可用的统计数据，并将 UP 主、粉丝、上传时间和六项互动统计排成 3×3 Description 网格；缺失项不显示。

## 视频与字幕

provider registry 只负责视频站点来源，不参与模型配置。Bilibili 保持现有视频信息和 `upName` 兼容行为。YouTube 支持页面 `captionTracks`、同源 youtubei fallback、JSON3/XML 字幕、人工/自动/翻译字幕和可选官方 API 元数据配置。

## 模型配置

设置页不包含模型供应商概念。文本模型与图片模型各自保留一组独立配置：

- Base URL
- API Key
- 模型
- 连通性测试

请求继续使用 OpenAI-compatible 协议。文本请求自动优先流式 fetch，必要时回退 GM XHR；图片请求使用现有自动通道。API Key 优先保存在 Tampermonkey storage，不回填密码输入框，粘贴 `Bearer <key>` 时会去掉前缀。

发布构建不得内嵌 API Key、OAuth Token、私钥或其他用户凭据；本地 `.env.*` 文件只允许用于
显式集成测试，并保持 Git 忽略，生产代码不得通过 `import.meta.env` 或等价路径读取这些值。

旧配置加载时保留已有文本/图片 URL、Key 和模型值，但丢弃供应商、模型预设列表、远程模式和一图流模式等废弃字段。

图片配置另有生图方式选择：

- `api`：使用图片 Base URL、API Key 和模型，作为默认模式。
- `chatgpt_web`：使用 `https://chatgpt.com/g/g-p-.../project` 根页，每次在该 Project 内新建独立聊天，不要求图片 API 配置。

两种模式的配置值分别保留。网页模式连通性测试只检查接收页 heartbeat，不发送生图请求。

设置页只显示当前生图方式对应的字段。摘要页顶部固定显示视频卡片和模型配置；字幕选择合并到配置行的字幕胶囊，切换只更新 `selectedSubtitleId`，等待用户主动重新生成。只有总结和后续消息进入滚动区。初始总结流式更新不得替换滚动容器；无完整总结时禁用输入框并显示“开始总结”，完整总结生成后才显示透明、左对齐的功能工具栏。最新完成的 Agent 回复显示三个本地固定快捷意图，点击后走现有问答发送路径。

设置页的“通用”分组包含打开视频时自动提取并生成，以及一个紧凑 Radio Chip Group 形式的全局视频总结预设。选项为极简白话、结构化笔记、批判分析、行动清单、时间轴定位和自定义；首次选择自定义时展开 textarea，保存后再次进入设置默认折叠，内容继续保留。空自定义 Prompt 不允许保存。“语言”分组包含界面语言和“偏好字幕获取及总结语言”。偏好语言默认中文，仅用于从当前视频可用字幕中预选轨道，摘要页字幕胶囊中的具体轨道下拉始终保留。ChatGPT Project 地址的根页提示以内联小字显示在字段标题后。

摘要运行时使用 `defaultPrompts.v2.ts`，`defaultPrompts.ts` 保持不变用于效果对比。短视频直接执行所选预设；长视频先生成有序事实清单，再由合并 Prompt 执行所选目标预设。时间轴预设把结构化字幕格式化为真实的 `[开始-结束] 文本`，禁止模型猜测时间。预设模板指纹进入缓存键和缓存匹配，内置模板升级或自定义文本变化都会隔离旧摘要。

摘要、后续问答、生图和模型连通性测试所使用的运行时 Prompt 文本全部由 `defaultPrompts.v2.ts` 导出，调用模块只负责变量填充和请求。设置页不用系统原生 `<select>` 弹出层，而使用可键盘关闭的内嵌 listbox；菜单在字段下方参与布局并滚动到可见区域，箭头距控件右侧 12px。主体 Panel 按钮不继承全局 hover 背景，右侧导航按钮保留独立 hover 状态。

ChatGPT 网页生图的成功结果和图片分块通过不同 GM storage key 传播。视频页收到成功结果后必须短时等待全部声明分块，不得因跨 key 通知乱序立即失败。快捷功能按钮使用无边框、透明背景的纯图标样式，与对话背景融合。

生成图在对话内使用可点击预览组件。点击缩略图后在 Shadow Root 顶层打开全屏遮罩，底部操作栏提供缩小、比例、放大、左右旋转、恢复适配尺寸和下载；图片支持滚轮缩放与拖动，并可通过右上关闭、Esc 或点击遮罩退出。视频卡片统计项 hover 保持默认箭头光标，摘要快捷工具 tooltip 显示在按钮右下方。

## 意图与图片生成

当用户在对话框中表达了生成图片的意图（如“画图”、“生图”等），系统会自动拦截纯文本对话，改为调用图片模型。
本地使用以下固定模板拼接摘要，调用图片生成接口：

```text
根据以下视频内容总结，生成一张信息可视化的精美配图，风格清晰美观，适合作为视频总结的封面图：

{summary}
```

- 不调用文本模型生成图片 Prompt。
- 不使用 HTML/CSS 卡片排版合成。

### ChatGPT 网页桥接

同一 userscript 同时运行在视频页和 ChatGPT 页。视频页通过版本化 GM storage job 发布最终图片 Prompt；匹配 Project 根页的 ChatGPT 接收端每次创建独立聊天、自动填入输入框并发送，只监听该任务聊天中新增的 assistant 图片。视频页成功组装图片并发布消费 ACK 后，接收端才返回 Project 根页；失败或 ACK 超时则保留当前结果聊天。

接收端优先把同源生成图下载为 Blob，以 512 KiB Base64 分块写回 GM storage；视频页重组为 `GeneratedImage` 后清理临时分块并进入现有图片缓存。无法下载二进制时可回传 HTTPS 图片 URL。heartbeat 30 秒失效，任务默认十分钟过期；网页模式失败不会自动调用图片 API。

若原生 GM value-change listener 不可用，桥接使用 `GM_getValue` 轮询回退。生图终态错误必须同时写入助手消息和顶部通知。

UI 采用原生 Shadow DOM 构建，采用极致简约的单栏全高设计，只保留摘要对话流和右上角设置入口。面板宽度可拖动调节，支持拖拽宽度、临时收起、悬浮按钮拖动和浏览器全屏隐藏。设置页有未保存更改时，切换 Tab 会先确认保存。
所有设置通过 Tampermonkey GM_setValue 存储。摘要与生成图片使用独立本地缓存；对话历史仅保留在当前页面会话中。一键清除缓存会清空摘要、图片及当前对话状态。

## 已知边界

- 无字幕视频不会自动 ASR。
- fetch 流式请求要求 API 支持浏览器 CORS；GM XHR fallback 可能一次性返回。
- 图片 API 必须返回现有客户端支持的 URL 或 Base64 字段。
- ChatGPT 网页模式依赖页面 DOM 和用户已有登录态，属于实验功能；页面改版、额度不足或生成拒绝会返回明确错误。
- 本项目不包含登录、支付、自建后端、云端历史或 playlist 批量总结。
