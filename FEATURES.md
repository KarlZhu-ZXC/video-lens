# V1 功能规格

## 1. 产品形态

V1 是一个运行在 Tampermonkey 中的 Bilibili 视频总结面板。用户打开 Bilibili 视频页后，页面右侧先出现可拖动的 mascot 悬浮启动按钮；点击后展开完整面板。面板提供 4 个 Tab：

- 摘要
- 视频洞察
- 一图流
- 设置

V1 的目标是让用户在一个面板内完成：

1. 读取当前视频字幕。
2. 生成流式 Markdown 摘要。
3. 查看低干扰模型思考状态。
4. 基于摘要继续进行视频洞察问答。
5. 生成一图流并导出 PNG。
6. 配置文本模型和图片模型。

V1 不包含账号系统、云端历史、支付、评论总结、弹幕总结、多平台支持或 Chrome Extension 版本。

## 2. 面板整体

### 2.1 面板位置

默认以页面右侧的悬浮启动按钮进入。完整面板可显示在页面左侧或右侧。

面板通过 Shadow DOM 注入，避免被 Bilibili 页面样式污染。

### 2.2 面板头部

完整面板头部内容：

- 标题：当前 Tab 名称，例如 `AI总结`、`视频洞察`、`一图流`、`设置`
- 副标题：当前 Tab 的说明文案
- 按钮：`收起`

### 2.3 收起状态

点击 `收起` 后，完整面板隐藏，页面上显示一个独立悬浮 mascot 按钮：

- 点击按钮：展开完整面板
- 拖动按钮：调整悬浮位置

折叠状态不持久化。刷新页面后默认从悬浮按钮开始，避免面板遮挡视频页面。

### 2.4 侧边 Tab

面板包含 4 个 Tab：

- `摘要`
- `视频洞察`
- `一图流`
- `设置`

点击 Tab 只切换面板内容，不刷新页面，不重新读取视频。

### 2.5 全局状态

面板内会显示状态文本，用于反馈当前任务：

- `等待操作`
- `读取视频信息`
- `已识别视频`
- `获取字幕`
- `生成摘要`
- `分块摘要 x/y`
- `合并长视频摘要`
- `摘要已生成`
- `请先生成摘要`
- `Video Insights`
- `已回答`
- `生成一图流 JSON`
- `校验一图流 JSON`
- `生成图片 prompt`
- `生成 AI 背景图`
- `渲染中文信息图`
- `一图流已生成`
- `PNG 已导出`
- 错误信息

完成、错误和关键状态会通过全局顶部通知提示。通知独立于 Tab 内容渲染，切换 Tab 不会重启动画。

## 3. 摘要 Tab

### 3.1 目标

摘要 Tab 是默认 Tab。它负责读取当前视频字幕，并生成可阅读的 Markdown 摘要。

### 3.2 显示内容

摘要 Tab 包含：

- 视频信息卡
- 模型配置、字幕语言和缓存时间
- 当已加载多个字幕源时，显示字幕源选择器
- 状态文本
- 摘要输出区
- 底部吸底操作按钮区

### 3.3 按钮

#### 开始总结 / 重新生成

按钮文案：`开始总结`；已有摘要时显示 `重新生成`。

点击后执行：

1. 读取当前视频信息。
2. 获取可用字幕。
3. 如果用户选择过字幕源，使用该字幕源；否则根据“字幕获取及总结语言”选择字幕：中文优先 `zh-CN -> zh -> first`，英文优先 `en -> fallback`。当总结语言变化时，会清除旧的自动字幕选择，下一次生成重新按新语言选择。
4. 根据字幕长度决定直接摘要或分块摘要。
5. 调用设置中的文本模型。
6. 以 stream 模式接收模型输出。
7. 实时更新摘要输出区。
8. 完成后写入本地缓存。

如果没有配置文本模型 API Key，显示错误：

```text
请先在设置中填写文本模型 API Key
```

如果当前视频没有公开字幕，显示错误：

```text
当前视频没有公开字幕
```

#### 复制

按钮文案：`复制`

点击后：

- 如果已有摘要，把摘要正文复制到剪贴板。
- 如果没有摘要，不执行操作。
- 复制成功后状态显示 `摘要已复制`。
- 复制失败时状态显示 `摘要复制失败：...`。

#### 清除此视频缓存

点击后：

- 删除当前视频、当前 prompt 和当前文本模型对应的摘要缓存。
- 清空当前面板中的摘要状态。
- 状态显示 `已清除此视频缓存`。

#### 清空全部缓存

点击后：

- 清空摘要缓存、一图流缓存和图片缓存。
- 清空当前面板中的摘要和一图流状态。
- 状态显示 `已清空全部缓存`。

### 3.4 摘要输出区

未生成摘要时显示：

```text
还没有摘要。
```

生成摘要时：

- 内容随 stream delta 实时追加。
- 不等待请求结束才显示。
- 输出使用 Markdown 渲染。

生成完成后：

- 显示最终摘要。
- 摘要会缓存到本地。
- 再次打开同一视频时，如果缓存命中，状态显示 `已载入缓存摘要`。

### 3.5 Markdown 渲染

摘要正文不是纯文本输出。V1 支持以下 Markdown 子集：

- `#` 到 `####` 标题
- 段落
- 无序列表
- 有序列表
- `**粗体**`
- 行内代码
- HTTP/HTTPS 链接
- HTTP/HTTPS 图片；HTTP 图片 URL 会优先升级为 HTTPS，减少 Mixed Content warning

渲染器通过 DOM 节点构建，不直接插入模型返回的 HTML。

### 3.6 模型思考

如果模型返回思考内容，V1 会按生成阶段低干扰展示：

- 只有思考、还没有正文时，显示一行 `模型思考` 状态和最新思考片段。
- 正文开始流式输出后，隐藏模型思考，避免打断摘要阅读。
- 生成完成后，显示一个默认收起的 `模型思考` 折叠行，用户可以手动展开查看完整内容。

支持的思考来源：

- `reasoning_content`
- `reasoningContent`
- `thinking`
- 正文里的 `<think>...</think>`

正文中的 `<think>...</think>` 不应该直接显示在摘要里。V1 会把它抽取为模型思考内容，再按上述规则展示。

## 4. 视频洞察 Tab

### 4.1 目标

视频洞察 Tab 允许用户基于当前视频摘要继续提问。

### 4.2 前置条件

用户必须先生成摘要。

如果没有摘要就提问，状态显示：

```text
请先生成摘要
```

### 4.3 显示内容

视频洞察 Tab 包含：

- 历史问答区
- 问题输入框
- 提交按钮
- 状态文本

### 4.4 可填写内容

#### 问题输入框

用户可以输入任意自然语言问题，例如：

- `这个视频最核心的观点是什么？`
- `有没有标题党？`
- `帮我提炼成行动清单。`
- `这个 UP 主的论据靠谱吗？`

空问题不会触发请求。

### 4.5 按钮

#### 提问

点击后执行：

1. 校验问题非空。
2. 校验已有摘要。
3. 将用户问题加入历史。
4. 调用文本模型回答。
5. 将回答追加到历史。
6. 状态显示 `已回答`。

### 4.6 模型上下文

视频洞察请求会使用：

- 当前视频标题
- 已生成摘要
- 视频字幕
- 最近的问答历史
- 用户当前问题

## 5. 一图流 Tab

### 5.1 目标

一图流 Tab 把摘要转成适合分享的中文信息图，并支持导出 PNG。

### 5.2 前置条件

一图流依赖摘要。

如果用户没有先生成摘要，点击生成一图流时会先尝试恢复本地缓存摘要。没有可用缓存时状态显示 `请先生成摘要`，不会自动重新跑摘要流程。

### 5.3 显示内容

一图流 Tab 包含：

- 生成模式配置
- 状态文本
- 信息图预览区
- 底部吸底操作按钮区

### 5.4 按钮

#### 生成一图流

点击后执行：

1. 确认已有摘要；没有则先尝试恢复缓存摘要。
2. 调用文本模型生成一图流 JSON。
3. 校验 JSON schema。
4. 按设置中的一图流模式决定是否生成 AI 背景图。
5. 使用当前模板在前端合成信息图。
6. 在预览区显示结果。

状态会按流程变化：

- `生成一图流 JSON`
- `校验一图流 JSON`
- `生成图片 prompt`
- `生成 AI 背景图`
- `渲染中文信息图`
- `一图流已生成`

#### 导出 PNG

点击后执行：

1. 如果已有一图流预览，直接导出。
2. 如果没有预览，先生成一图流。
3. 使用 `html-to-image` 按设置中的导出倍率生成 PNG base64 data URL。
4. 将 data URL 转为 Blob，再触发浏览器下载，提升 Tampermonkey 环境下的文件兼容性。
5. 文件名使用视频标题，非法文件名字符替换为 `_`。
6. 状态显示 `PNG 已导出`。

### 5.5 一图流数据

一图流 JSON 支持：

- 标题
- 副标题
- 核心结论
- 关键点
- 时间线
- takeaways
- 标签
- 来源信息

### 5.6 模板

V1 内置模板：

- `classic`
- `dense`
- `poster`

当前 UI 暂不暴露模板切换控件，默认使用配置中的模板。

### 5.7 一图流模式

一图流模式在设置 Tab 中选择：

- `只生成文字卡`
- `AI 背景 + 中文排版`
- `AI 图片模式`

一图流预览区带缩放工具。首次打开已有预览时会按面板宽度自动 AutoFit，避免 900px 信息图在侧边面板中显示过大。

## 6. 设置 Tab

### 6.1 目标

设置 Tab 用于配置语言、文本模型、图片模型和一图流模式。

### 6.2 字段

#### 界面显示语言

控件类型：下拉框

选项：

- `中文`
- `English`

用途：

- 控制面板 UI 文案语言。

#### 字幕获取及总结语言

控件类型：下拉框

选项：

- `中文`
- `English`

用途：

- 控制摘要、长视频分块/合并、视频洞察和一图流 JSON 使用的 prompt 语言。
- 选择英文时，Bilibili 字幕源会优先尝试英文字幕；没有英文字幕时回退到可用字幕。

#### 文本供应商

控件类型：下拉框

当前选项：

- `Minimax-CN`

用途：

- 决定文本模型列表。
- 决定 Base URL 的默认值。
- 后续可扩展更多供应商。

#### Base URL

控件类型：文本输入框

用途：

- 配置 OpenAI-compatible 文本模型 base URL。

默认值：

```text
https://api.minimax.io/v1
```

保存规则：

- 保留用户填写的域名。
- 去掉尾部 `/`。
- 如果用户填写完整 `/chat/completions`，保存时转回 base URL。

请求规则：

- 发起文本请求时自动拼接 `/chat/completions`。

示例：

```text
https://api.minimax.io/v1
https://api.minimaxi.com/v1
https://my-proxy.example.com/minimax/v1
```

#### API Key

控件类型：密码输入框

用途：

- 配置文本模型 API Key。

支持填写：

```text
sk-...
Bearer sk-...
```

保存规则：

- 自动去掉开头的 `Bearer `。
- 自动 trim 首尾空格。
- 已保存的 API Key 不会回填到密码输入框。
- 输入框留空保存时继续使用旧 API Key。
- 配置优先写入 Tampermonkey storage；如果没有 GM storage 能力，fallback 写入 `localStorage` 前会剥离 API Key。

请求规则：

- 请求时添加：

```text
Authorization: Bearer <API Key>
```

#### 文本请求通道

控件类型：无 UI 控件

用途：

- 项目统一使用自动请求通道。
- 流式文本请求优先走浏览器 `fetch`；失败且 Tampermonkey 能力可用时回退到 `GM_xmlhttpRequest` 的非流式请求。
- 不再让用户在设置页手动选择 Fetch / GM XHR。

#### 模型

控件类型：下拉框

当前选项：

- `MiniMax-M2.7`
- `MiniMax-M2.5`
- `MiniMax-M2.1`
- `MiniMax-M2.1-highspeed`
- `MiniMax-M2`
- `MiniMax-M1`
- `MiniMax-Text-01`

用途：

- 控制摘要、视频洞察、一图流 JSON、图片 prompt 使用的文本模型。

#### 图片 API URL

控件类型：文本输入框

用途：

- 配置图片模型 API endpoint。

默认值：

```text
https://api.openai.com/v1/images/generations
```

V1 不对图片 API URL 做供应商级适配。

#### 生图模型启用状态

控件类型：无 UI 控件

用途：

- 生图模型默认启用，不再在设置页提供开关。
- 是否实际调用图片模型由一图流模式决定。

#### 图片 API Key

控件类型：密码输入框

用途：

- 配置图片模型 API Key。

保存规则：

- 已保存的 API Key 不会回填到密码输入框。
- 输入框留空保存时继续使用旧 API Key。

#### 图片请求通道

控件类型：无 UI 控件

用途：

- 图片生成请求统一使用自动请求通道。
- 优先走浏览器 `fetch`，失败后在可用时回退 `GM_xmlhttpRequest`。

#### 图片模型

控件类型：文本输入框

用途：

- 配置图片生成模型。

默认值：

```text
gpt-image-1
```

#### 一图流模式

控件类型：下拉框

选项：

- `只生成文字卡`
- `AI 背景 + 中文排版`
- `AI 图片模式`

用途：

- 控制一图流生成时是否调用图片模型，以及如何合成最终图片。
- 选择 `只生成文字卡` 时，生图字段会禁用，保存时不校验生图 API Key。

#### 面板宽度

控件类型：侧边 resize handle

用途：

- 控制摘要、视频洞察、一图流和设置 Tab 的共用面板宽度。
- 拖动面板侧边调整宽度，松开后写入配置；刷新后保持上次宽度。拖拽保存不会弹出 `设置已保存` toast。
- 当前拖拽范围为 `300px` 到 `900px`。
- 设置页不再提供面板宽度下拉框。

### 6.3 按钮

#### 连通性测试（文本模型）

位置：文本模型设置区域 header 右侧。

按钮 tooltip 会说明该操作会实际发送一次轻量 API 请求。点击后先检查本地配置，再使用当前文本模型发起一次简单 chat completions 请求。

可能状态：

- `请先在设置中填写文本模型 API Key`
- `请填写文本模型名称`
- `请填写文本模型 Base URL`
- `测试文本模型连通性`
- `文本模型连通性正常`
- 失败时显示具体错误信息，并触发 toast

#### 连通性测试（生图模型）

位置：生图模型设置区域 header 右侧。

按钮 tooltip 会说明该操作会实际发送一次轻量 API 请求。点击后先检查本地配置，再使用当前图片模型发起一次简单图片生成请求。

可能状态：

- `当前一图流模式不需要生图模型`
- `请先在设置中填写图片模型 API Key`
- `请填写生图模型名称`
- `请填写生图模型 API URL`
- `测试生图模型连通性`
- `生图模型连通性正常`
- 失败时显示具体错误信息，并触发 toast

#### 保存设置

点击后执行：

1. 读取当前表单字段。
2. 规范化文本模型 Base URL。
3. 规范化文本模型 API Key。
4. 保存界面语言和字幕/总结语言。
5. 保存图片模型配置。
6. 保存一图流模式。
7. 强制文本和图片请求模式为自动，强制生图模型启用。
8. 状态显示 `设置已保存`。

保存后配置优先写入 Tampermonkey storage。没有 GM storage 能力时，配置可 fallback 到浏览器 `localStorage`，但 API Key 会被剥离。

## 7. 本地缓存

### 7.1 摘要缓存

摘要缓存 key 包含：

- 视频 source id
- prompt id
- 文本模型

因此：

- 同一视频重复打开可复用缓存。
- 切换模型后会生成新的摘要缓存。
- 切换 prompt 后会生成新的摘要缓存。

### 7.2 一图流缓存

一图流相关数据保存在本地缓存中，用于减少重复生成。

### 7.3 图片缓存

AI 背景图结果可缓存在本地，避免重复生成同一图像。

### 7.4 缓存管理

摘要 Tab 提供两个缓存管理入口：

- `清除此视频缓存`：只清理当前视频摘要缓存。
- `清空全部缓存`：清理摘要、一图流和图片缓存。

## 8. 错误处理

V1 通过状态文本向用户展示错误。

常见错误：

- 当前视频没有公开字幕
- 字幕列表请求失败
- 字幕请求失败
- 请先在设置中填写文本模型 API Key
- Text generation failed
- Image generation failed
- 摘要复制失败
- 一图流 JSON 校验失败

错误不会使页面崩溃；面板继续保留，用户可以调整设置后重试。

## 9. 技术边界

当前是带 Tampermonkey grant 的 userscript：

- 使用 `GM_getValue` / `GM_setValue` 保存配置
- 使用 `GM_xmlhttpRequest` 支持跨域模型请求
- 使用 `GM_setClipboard` 复制摘要
- 不声明 `@connect`

因此：

- `fetch` 模式下 AI API 必须支持浏览器 CORS。
- `gm_xhr` 模式下可以绕过浏览器 CORS，但文本请求按非流式响应处理。
- API Key 保存在本机 Tampermonkey storage；它不提供账号级加密、跨设备同步或远程密钥托管。
- 如果要支持多端同步、远程密钥托管或后台任务，应迁移到 Chrome Extension 或增加后端代理。

## 10. UI 与资源安全

### 10.1 视觉风格

当前 UI 遵循 `DESIGN.md` 中的 Obsidian 高对比暗色方向：

- zinc 灰阶 surface scale
- 高对比正文和次级文本
- 以 border 分层为主，减少重阴影
- 主操作使用底部吸底按钮区
- 顶部全局通知不随 Tab 内容重渲染

### 10.2 外部资源 URL

为了减少 HTTPS 页面中的 Mixed Content warning，V1 会对显示型资源 URL 做规范化：

- `//example.com/image.png` 转为 `https://example.com/image.png`
- `http://example.com/image.png` 优先升级为 `https://example.com/image.png`
- `data:` URL 和已有 HTTPS URL 保持不变

该规则用于视频封面、Markdown 图片、一图流图片和背景图。它不绕过 CORS，也不改变文本/图片模型 API 的请求地址。
